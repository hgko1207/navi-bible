# 내비따라성경읽기 앱/웹 리서치

## 1. 현재 상황 분석

### 콘텐츠 구조 (마크다운 파일 기반)

| 항목           | 내용                                                      |
| -------------- | --------------------------------------------------------- |
| 프로그램명     | 내비따라성경읽기 (1년3독)                                 |
| 일정           | 월~금, 약 4개월에 1독 (연 3독)                            |
| 구약           | 1일차 ~ 약 48일차                                         |
| 신약           | 약 49일차 ~ 약 62일차+                                    |
| 일차별 구성    | 요일, 성경 범위, 핵심 포인트, 요약 해설, 유튜브 음원 링크 |
| 유튜브 음원    | 개역개정 성경 낭독 음원 (속도 조절 가능)                  |
| 현재 배포 방식 | 카카오톡 단톡방 공지                                      |

### 일차별 데이터 구조 (파일에서 추출)

```
- 제목: "내비따라성경읽기 (1년3독)"
- 요일 + 일차: "월. 1일차"
- 성경 범위: "창1-19장"
- 핵심 포인트: 구절별 키워드 (예: "창1:1-25 ⇒ 자연과학 창조")
- 본문 요약: 해당 범위의 신학적 해설 (200~400자)
- 참고: "구약읽기 내비게이션 p61-72 참고하세요"
- 유튜브 링크: 개역개정 음원 URL
- 안내 문구: 데이터 요금, 속도 조절 안내
```

### 현재 불편한 점

1. 매일 카톡 단톡방에 들어가서 공지를 확인해야 함
2. 이전 일차 내용을 다시 보려면 스크롤해서 찾아야 함
3. 진도 관리 (어디까지 들었는지) 기능 없음
4. 개인 기록/메모 기능 없음
5. 전체 일정을 한눈에 볼 수 없음

## 2. 웹 vs 앱 비교 분석

### 네이티브 앱 (React Native / Flutter)

| 장점               | 단점                               |
| ------------------ | ---------------------------------- |
| 푸시 알림 안정적   | 개발 기간 길고 비용 높음           |
| 오프라인 지원 우수 | iOS/Android 각각 스토어 심사 필요  |
| 네이티브 UX        | 업데이트 시 스토어 재심사          |
|                    | 사용자가 앱 설치해야 함 (진입장벽) |
|                    | 1인 개발 시 유지보수 부담 큼       |

### 웹앱 (일반 웹사이트)

| 장점                    | 단점                              |
| ----------------------- | --------------------------------- |
| 링크 하나로 공유 가능   | 푸시 알림 제한적 (iOS)            |
| 설치 불필요             | 오프라인 지원 약함                |
| 배포/업데이트 즉시 반영 | 홈 화면 아이콘 없으면 접근성 낮음 |
| 개발 비용 낮음          |                                   |
| 모든 기기에서 동작      |                                   |

### PWA (Progressive Web App) - **추천**

| 장점                                | 단점                           |
| ----------------------------------- | ------------------------------ |
| 웹 + 앱의 장점 결합                 | iOS 푸시 알림은 iOS 16.4+ 필요 |
| 홈 화면에 설치 가능                 | 일부 네이티브 기능 제한        |
| 오프라인 캐싱 지원 (Service Worker) |                                |
| 링크로 공유 가능                    |                                |
| 스토어 심사 불필요                  |                                |
| 빠른 개발 및 배포                   |                                |
| 업데이트 즉시 반영                  |                                |

### 결론: **PWA (Progressive Web App) 추천**

**이유:**

1. **콘텐츠 특성** — 텍스트 요약 + 유튜브 임베드가 핵심. 네이티브 기능(카메라, GPS 등)이 불필요
2. **배포 용이성** — 단톡방에 URL 하나만 공유하면 끝. 앱 설치 강요 불필요
3. **개발 효율** — 1인 개발 기준 웹 기술만으로 충분. 코드 하나로 모든 플랫폼 대응
4. **유지보수** — 콘텐츠 업데이트가 즉시 반영됨. 스토어 심사 대기 없음
5. **확장성** — 나중에 사용자가 많아지면 네이티브 앱으로 전환 가능

### PWA (Progressive Web App) 란?

**한마디로: "웹사이트인데 앱처럼 쓸 수 있는 것"**

일반 웹사이트를 만들되, 특별한 기술(Service Worker, Web App Manifest)을 추가해서 **네이티브 앱처럼 동작**하게 만든 것입니다.

#### 아이폰 & 갤럭시 모두 지원됩니다

| 기능 | 아이폰 (Safari) | 갤럭시 (Chrome) |
|------|-----------------|-----------------|
| 홈 화면에 설치 | ○ (공유 → 홈 화면에 추가) | ○ (설치 배너 자동 표시) |
| 앱처럼 실행 (전체화면) | ○ | ○ |
| 오프라인 사용 | ○ (캐시된 콘텐츠) | ○ |
| 푸시 알림 | ○ (iOS 16.4+, 2023년~) | ○ |
| 앱 스토어 필요 | ✕ (필요 없음) | ✕ (필요 없음) |

#### 사용자 입장에서 이렇게 동작합니다

```
1. 카톡 단톡방에 URL 공유 (예: https://navi-bible.vercel.app)
2. 링크를 눌러서 웹사이트 접속
3. "홈 화면에 추가" 하면 앱 아이콘이 생김
4. 이후에는 아이콘 터치 → 앱처럼 바로 실행 (주소창 없이 전체화면)
5. 매일 접속하면 오늘의 말씀이 바로 표시됨
```

#### 왜 네이티브 앱 대신 PWA가 더 좋은가?

- **앱 설치 강요 없음** — "앱스토어 가서 다운받으세요"가 아니라 링크만 보내면 됨
- **아이폰/갤럭시 동시 지원** — 코드 하나로 모든 기기에서 동작
- **업데이트 자동** — 사용자가 앱 업데이트할 필요 없음, 웹만 수정하면 즉시 반영
- **비용 0원** — 앱스토어 개발자 등록비(Apple 연 $99, Google $25) 불필요
- **핸드폰 사용에 최적** — 모바일 퍼스트로 설계하면 앱과 사용감 동일

> 결론: 아이폰 + 갤럭시 사용자 모두에게 **링크 하나로 배포**할 수 있고, 홈 화면에 추가하면 **앱과 동일한 경험**을 제공합니다. 이 프로젝트의 사용 케이스(텍스트 읽기 + 유튜브 듣기)에 가장 적합한 선택입니다.

## 3. 기술 스택 리서치

### 프론트엔드 프레임워크 비교

| 프레임워크          | 적합도 | 이유                                                |
| ------------------- | ------ | --------------------------------------------------- |
| **Next.js (React)** | ★★★★★  | SSR/SSG 지원, PWA 플러그인 풍부, Vercel 무료 배포   |
| Vite + React        | ★★★★☆  | 빠른 개발, 가볍지만 SSR 직접 구성 필요              |
| Nuxt.js (Vue)       | ★★★★☆  | Vue 기반, Next.js와 유사한 장점                     |
| Astro               | ★★★☆☆  | 정적 사이트에 최적이나 인터랙티브 기능 추가 시 복잡 |
| 순수 HTML/CSS/JS    | ★★☆☆☆  | 가능하지만 상태관리, 라우팅 직접 구현 필요          |

### 추천 스택: **Next.js + TypeScript + Tailwind CSS**

```
프론트엔드: Next.js 14+ (App Router)
스타일링:   Tailwind CSS
언어:       TypeScript
PWA:        next-pwa 또는 @serwist/next
배포:       Vercel (무료 티어)
데이터:     마크다운 파일 → JSON 변환 (빌드 타임)
상태관리:   localStorage (진도 체크, 메모 등)
```

### 배포 옵션

| 플랫폼           | 무료 티어 | 특징                                        |
| ---------------- | --------- | ------------------------------------------- |
| **Vercel**       | ○         | Next.js 공식 지원, 자동 배포, 커스텀 도메인 |
| Netlify          | ○         | 정적 사이트 최적, 간편한 배포               |
| Cloudflare Pages | ○         | 글로벌 CDN, 빠른 속도                       |
| GitHub Pages     | ○         | 정적 사이트만 가능                          |

## 4. 핵심 기능 정의

### MVP (최소 기능 제품)

1. **오늘의 말씀** — 현재 일차의 요약 + 유튜브 임베드 자동 표시
2. **일차 목록** — 전체 일차를 리스트로 보기 (구약/신약 구분)
3. **유튜브 임베드** — 페이지 내에서 바로 재생 가능
4. **진도 체크** — 들은 일차 체크 표시 (localStorage)
5. **반응형 디자인** — 모바일 최적화 (주 사용 환경이 스마트폰)
6. **PWA 설치** — 홈 화면 추가 가능

### 확장 기능 (v2)

7. **개인 메모** — 일차별 메모/기도제목 기록
8. **진도율 대시보드** — 전체 진도 시각화 (프로그레스 바)
9. **알림** — 매일 정해진 시간에 알림 (PWA Push 또는 브라우저 알림)
10. **독서 통계** — 연속 일수, 완독 횟수 등
11. **공유 기능** — 오늘의 말씀을 카톡/SNS로 공유
12. **다크 모드** — 밤 시간 읽기 편의

### 확장 기능 (v3 - 커뮤니티)

13. **그룹 기능** — 함께 읽는 그룹 생성, 그룹원 진도 확인
14. **관리자 모드** — 새 일차 콘텐츠 등록/수정 (CMS)
15. **댓글/나눔** — 일차별 말씀 나눔 게시판

## 5. 데이터 설계

### 마크다운 → 구조화 데이터 변환

현재 마크다운 파일에서 추출 가능한 데이터:

```typescript
interface DailyReading {
  day: number; // 일차 (1, 2, 3...)
  weekday: string; // 요일 (월, 화, 수...)
  testament: '구약' | '신약'; // 구약/신약 구분
  bibleRange: string; // 성경 범위 (예: "창1-19장")
  keyPoints: {
    // 핵심 포인트
    verse: string; // 구절 (예: "창1:1-25")
    summary: string; // 요약 (예: "자연과학 창조")
  }[];
  content: string; // 본문 요약 해설
  reference: string; // 내비게이션 참고 페이지
  youtubeUrl: string; // 유튜브 음원 링크
  youtubeId: string; // 유튜브 영상 ID (임베드용)
}
```

### 데이터 저장 방식

- **콘텐츠 데이터**: 빌드 타임에 마크다운 → JSON 변환 (정적 데이터)
- **사용자 데이터**: localStorage (진도, 메모 등 — 서버 불필요)
- **v3 이후**: 필요 시 Supabase 또는 Firebase로 백엔드 추가

## 6. UI/UX 방향

### 디자인 컨셉

- **심플하고 따뜻한** — 성경 읽기 앱답게 차분한 색상 (베이지, 네이비, 골드 포인트)
- **모바일 퍼스트** — 주 사용 환경이 스마트폰
- **큰 글씨, 넉넉한 여백** — 다양한 연령대 사용자 고려
- **원탭 접근** — 오늘의 말씀까지 최소 클릭

### 주요 화면

1. **홈 (오늘의 말씀)** — 오늘 읽을 내용 + 유튜브 플레이어
2. **캘린더/목록** — 전체 일정 보기, 완료 표시
3. **상세 페이지** — 각 일차별 요약 + 유튜브 + 메모
4. **내 기록** — 진도율, 메모 모아보기

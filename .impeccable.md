## Design Context

### Users
한국어 사용 그리스도인. 이동 중(대중교통 등)이나 저녁 시간에 사용. 연령대 다양 — 20대부터 60대 이상까지. 한 손으로 스마트폰을 쥐고 짧은 집중 시간에 읽는 경우가 많으므로, 빠르게 오늘 분량을 찾아 재생·읽기를 시작할 수 있어야 한다. 저녁 사용이 많아 다크모드가 중요하다.

### Brand Personality
**경건하고 따뜻하며 간결하다.**
- 경건: 말씀을 읽는 행위에 걸맞은 차분하고 집중적인 분위기
- 따뜻함: 차가운 기술 앱이 아닌, 매일 함께하는 친근한 동반자
- 간결: 불필요한 요소 없이 콘텐츠(말씀)에 집중

감정 목표: 앱을 열었을 때 "오늘도 읽을 수 있겠다"는 차분한 동기부여. 완료 체크 시 작은 성취감.

### Aesthetic Direction
- **색상 팔레트**: 웜 스톤/앰버 기조. Primary accent `#d97706` (amber-600). 구약은 amber/orange 계열, 신약은 blue 계열로 구분. 배경은 따뜻한 오프화이트 `#f8f6f3`.
- **다크모드**: 필수. 완전한 검정이 아닌 웜 다크(`#0c0a09`, `#1c1917`).
- **타이포**: 시스템 폰트(-apple-system, Noto Sans KR). 본문 가독성 최우선, 글꼴 크기 조절 지원.
- **레이아웃**: 세로 스크롤 단일 컬럼. 카드 기반 UI, 둥근 모서리(rounded-2xl~rounded-[28px]).
- **복잡함 금지**: 모달·팝업·탭 과다 사용 금지. 한 화면에 한 가지 핵심 작업.
- **애니메이션**: fadeInUp 입장 애니메이션 외 과도한 모션 지양.

### Design Principles
1. **콘텐츠 우선** — UI 크롬은 최소화하고 말씀 텍스트와 오디오 플레이어가 화면의 중심이 되어야 한다.
2. **한 손 조작** — 하단 네비게이션, 큰 터치 타겟(min 44px), 주요 액션은 엄지 닿는 영역에.
3. **저조도 친화** — 다크모드를 2등 시민 취급 금지. 모든 컴포넌트 설계 시 라이트/다크 동시 고려.
4. **조용한 성취감** — 완료 체크, 스트릭, 진도 표시는 과하지 않게. 축하보다 지속을 돕는 방향.
5. **점진적 복잡도** — 핵심 기능(오늘 읽기)은 즉시 접근 가능. 메모·통계 등 부가 기능은 깊은 곳에.

### Planned Extensions (우선순위 순)
1. **읽기 알림** — 매일 정해진 시간 푸시 알림 (Web Push API)
2. **묵상 메모** — 일차별 짧은 개인 노트 저장 (localStorage)
3. **연속 읽기 스트릭** — 동기부여용 연속 일수 표시
4. **공유하기** — 오늘 말씀 범위/요약을 네이티브 공유 (Web Share API)
5. **읽기 통계** — 월별 달력 히트맵, 총 완독 횟수
6. **북마크** — 기억하고 싶은 일차 즐겨찾기

### Tech Stack
- Next.js 16 (App Router) + TypeScript + Tailwind CSS v4
- PWA (Service Worker, Web App Manifest)
- YouTube IFrame API (오디오 재생)
- Web Speech API (TTS)
- localStorage (모든 데이터 저장, 서버 없음)
- Vercel 배포

description: 사용자 요청사항을 플랜에 정리
disable-model-invocation: true
argument-hint: [요청사항]
---

사용자의 요청사항을 플랜에 정리합니다.

1. 사용자의 요청 내용을 분석합니다: $ARGUMENTS
2. 현재 플랜 파일을 읽습니다. (Claude plans 파일)
3. 요청사항을 "다음 작업 (구현 예정)" 섹션에 추가합니다.
   - 각 항목에 제목, 구현 내용, 목적을 명확히 작성합니다.
   - 이미 완료된 기능은 "완료된 기능" 섹션으로 이동합니다.
   - 중복 항목은 병합합니다.
4. 사용자에게 정리된 플랜을 간단히 요약합니다.
